Elements of Statistical Learning (Hastie, Tibshirani & Friedman) Chapter 4.2 discusses Linear Regression of an Indicator Matrix. In a 3 class problem similar to the one above, linear regression will "mask" classes when $K\ge p$, with $K:=$ #classes & $p:=$ #predictors. In order to demonstrate the class masking phenomenon in a 3 class problem then, it will be necessary to exclude the intercept in the model.

Quick derivation of the normal equation for linear regresion:

  1. Let $\hat{\textbf{Y}}=\boldsymbol{\beta}^T\textbf{X}$ be the fitted value. Then the objective is to minimize the squared distance between $\hat{\textbf{Y}}$ and $\textbf{Y}$:
    $(\boldsymbol{\beta}^T\textbf{X}-\textbf{Y})^T(\boldsymbol{\beta}^T\textbf{X}-\textbf{Y})$
  2. This is a convex optimization problem and the minimum is where the derivative is equal to 0:
    $\frac{\partial}{\partial\boldsymbol{\beta}}(\boldsymbol{\beta}^T\textbf{X}-\textbf{Y})^T(\boldsymbol{\beta}^T\textbf{X}-\textbf{Y})=2\textbf{X}^T\textbf{X}-2\textbf{X}^T\textbf{Y}=0$
  3. Rearrange and multiply both sides by $(\textbf{X}^T\textbf{X})^{-1}$, giving the normal equation:
    $\boldsymbol{\beta}=(\textbf{X}^T\textbf{X})^{-1}\textbf{X}^T\textbf{Y}$

What does this mean for the decision boundary? $$\begin{bmatrix}p_1(\textbf{X})\\p_2(\textbf{X})\\p_3(\textbf{X})\end{bmatrix}=\begin{bmatrix}\beta_{0,1} & \beta_{1,1}\\\beta_{0,2} & \beta_{1,2}\\\beta_{0,3} & \beta_{1,3}\end{bmatrix}\begin{bmatrix}petal\:length\\sepal\:length\end{bmatrix}$$

In this problem set up, there are 3 linear decision boundaries: $p_1(\textbf{X})=p_2(\textbf{X})$, $p_1(\textbf{X})=p_3(\textbf{X})$, and $p_2(\textbf{X})=p_3(\textbf{X})$. The first boundary can be determined as follows:

$$-0.362[petal\:length]+0.289[sepal\:length]=0.058[petal\:length]+0.018[sepal\:length]$$

Which can be simplified to the equation of a line: $$sepal\:length=1.550[petal\:length]$$

We can alternatively solve the regression problem using gradient descent instead of the normal equation in order to confirm our results.

Because the loss function is convex, updating the coefficients in the (negative) direction of the gradient will converge to the global minimum after a specified number of iterations. The graident of the loss function $L(\boldsymbol{\beta})=\frac{1}{n}(\textbf{X}^T\boldsymbol{\beta}-\textbf{Y})^2$ is:

$\frac{\partial L}{\partial\beta}=\frac{2}{n}\textbf{X}^T(\textbf{X}^T\boldsymbol{\beta}-\textbf{Y})$

The coefficients are iteratively updated in the direction of the gradient (adjusted by a learning rate $\alpha$). The $m$th iteration looks like this: $\boldsymbol{\beta}_{m}=\boldsymbol{\beta}_{m-1}-\frac{\alpha}{n}\textbf{X}^T(\textbf{X}^T\boldsymbol{\beta}-\textbf{Y})$

In [37]:
beta = np.zeros((len(X.columns),3))

for i in range(10000):
    beta = beta - (0.01/len(X))*np.dot(X.transpose(), (np.dot(X,beta)-Y))
beta

Note that alpha had to be chosen carefully because the observed response $\textbf{Y}$ can only take values between $0$ and $1$. Therefore a learning rate which is too large will diverge faster than the loss.

Our result from gradient descent confirms our earlier result from the normal equation. We can also attempt to confirm the results for a third time using $\texttt{LinearRegression}$ from the $\texttt{sklearn}$ package.

The above figure has been set up to demonstrate that the "probability function" for $versicolor$, $p_2(\textbf{X})$ (the orange dashed lined) is never larger than either of the other two class probability functions $p_1(\textbf{X})$ or $p_3(\textbf{X})$, even when the class is $versicolor$.

This is because of the setup of the response variable -the indicator variable that is $1$ when the species is $versicolor$ and $0$ otherwise. As can be seen from the points above, the OLS solution is a nearly horizontal line. In comparison to the "probability functions" for the other two classes below:

The OLS solutions to these two "probability functions" are lines with a slope $|m|>0$. Therefore the class probabilities for $p_1(\textbf{X}):=setosa$ and $p_2(\textbf{X}):=virginica$ will always be larger than that of $p_2(\textbf{X}):=versicolor$ and the second (masked) class will never be predicted.

As discussed earlier (from Elements of Statistical Learning), this can be solved by increasing the number of parameters in the model. The simplest solution is to add an intercept:
